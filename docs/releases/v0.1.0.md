# Lit Panel v0.1.0

This is the first public Lit Panel desktop release.

## Highlights

- A complete bilingual Lit Panel interface for library browsing, paper detail,
  AI reading, graphs, atoms, ideas, maps, and projects.
- A configurable PDF reading pipeline with independent **Update Graph** and
  **Update Ideas** controls.
- Safer import handling for PDFs, NBER IDs, DOI records, retries, and failed
  reading jobs.
- Library-scoped search and API access, including corrected SQLite FTS5 library
  filtering.
- Topic graph matching that avoids unrelated paper results and renders formal
  Cytoscape figures for valid topics and papers.
- A clean synthetic demo library containing 8 papers, 12 atoms, 3 ideas, and 18
  graph edges.
- A self-contained Apple Silicon desktop app with portable Python, standalone
  Next.js, local FastAPI services, and first-run demo-data initialization.

## Privacy And Packaging

- The release contains no private papers, working databases, API keys, `.env`
  files, build-machine paths, or Python bytecode caches.
- API provider records are included only as disabled templates with empty keys.
- The app stores writable data under `~/Library/Application Support/Lit Panel/`
  and does not modify signed application resources.
- Electron was upgraded to 43.1.0 and the desktop dependency audit reports zero
  known vulnerabilities.

## Validation

- Next.js production build and frontend lint passed.
- Backend search regression tests passed.
- The packaged app completed an end-to-end startup, search, demo-data, frontend,
  and graceful-shutdown smoke test.
- macOS code-signature verification and DMG checksum verification passed.

Download `Lit-Panel-0.1.0-arm64.dmg` from the assets below. The adjacent

## Known Limits

- This build supports macOS Apple Silicon (`arm64`) only.
- It is ad-hoc signed but not Apple-notarized. macOS may require opening the app
  once with Finder's **Open** context-menu action.
- AI operations send the selected paper text and prompt to the provider
  configured by the user; they are not offline operations.
